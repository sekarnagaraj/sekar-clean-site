How to add more later

This site is designed for continuous updates.

For a new skill, use case, architecture or blog, you only edit one file: content-data.js. Then push to GitHub and Vercel will redeploy automatically.

Add a new skill

  1. Open content-data.js.
  2. Find capabilities.
  3. Copy one existing object and change title, summary and proof points.
  4. Save, commit and push.

Add a new project / use case

  1. Open content-data.js.
  2. Find projects.
  3. Add business problem, solution, technologies, highlights and learning.
  4. It will appear automatically under Projects.

Add a new blog

  1. Open content-data.js.
  2. Find blogs.
  3. Copy the blog template from docs/ADD_CONTENT_GUIDE.md.
  4. Change title, category, summary, tags and body paragraphs.
git add .
git commit -m "Add new blog or project"
git push

After push, Vercel updates the live site automatically.
